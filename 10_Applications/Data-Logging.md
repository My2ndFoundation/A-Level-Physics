# Data-Logging

> Deliberately shallow **cross-subject orientation** page — how computer-based measurement supports A-Level practical physics, not a full instrumentation course.

## Problem Context

A data logger uses sensors and a computer to record measurements automatically at set time intervals. It captures fast or long experiments that are impossible to time by hand — e.g. acceleration of a falling card, or a slow cooling curve.

## Physical Ideas

- [[Measurement-Uncertainty]]

## Mathematical Tools

- Sampling interval choice; reading gradients/areas from logged graphs.

## Typical Questions

- State an advantage of data logging over manual timing for a fast event.
- Explain how sampling rate affects the detail captured.

## Method Outline

1. Connect sensors (motion, temperature, voltage) to the logger.
2. Set sampling rate and duration appropriate to the event.
3. Export to software; analyse via [[Line-of-Best-Fit-Graph]] and tables.

## Assumptions

- Sensors are calibrated; sampling rate is high enough to resolve the change.

## Links to Other Subjects

- Mathematics: data analysis, uncertainty, [[Combining-Uncertainties]].
- Computer Science: analogue-to-digital conversion, data storage.

## Frontier Links

- [[Semiconductor-Physics-Map]]

## Common Mistakes

- Sampling too slowly and missing rapid changes (aliasing).
- Assuming automated data is free of systematic error.
