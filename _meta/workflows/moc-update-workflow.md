# MOC Update Workflow

Rules for creating and updating Maps of Content.

- MOCs are navigation maps.
- MOCs should not duplicate atomic content.
- Add pages under correct headings.
- Preserve existing user edits when possible.
- Do not reorder unnecessarily.
- Use canonical wikilinks.
- Keep MOCs concise.
- MOCs may include short prerequisite routes and cross-topic links.
- Frontier MOCs should remain shallow and explicitly peripheral.

## Procedure

1. Identify the relevant MOC(s) for a new or updated page (by syllabus area, topic, and learning layer).
2. Locate the correct heading inside each MOC.
3. Add the canonical wikilink under that heading if not already present.
4. Leave unrelated sections and user edits untouched.
5. If a MOC heading does not exist for the page type, add the heading using the order from `_meta/templates/moc-template.md`.
6. Do not move existing links unless removing a genuine duplicate.
