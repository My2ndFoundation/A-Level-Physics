# Results Table

## Core Idea

A structured table that records measured and processed data so it can be read, checked, and turned into a [[Line-of-Best-Fit-Graph|graph]].

## Form

Rows of data under labelled columns. The independent variable is usually the first column, the dependent variable next, followed by repeats, means and any processed (e.g. log or squared) values.

## Axes / Labels / Components

- Every column heading is a **quantity and a unit**, separated by clear notation. Solidus form is preferred — `T / °C`, `v / m s⁻¹` — though `T (°C)` is also accepted.
- The quantity symbol is written in *italics*; the unit is upright.
- All raw readings of one quantity are given to the **same number of decimal places**, matching the instrument's resolution (see [[Significant-Figures-in-Measurements]]).
- Processed columns (means, logs) follow the [[Significant-Figures-in-Measurements|significant-figure]] rules — a mean may carry one extra figure.

## Physical Meaning

The table is the audit trail between the apparatus and the conclusion: it exposes anomalies, shows repeat consistency, and makes the data plottable.

## Gradient / Area / Intercepts

Not applicable directly — but the table supplies the coordinate pairs (and any linearised columns) used to extract a gradient or intercept from a [[Line-of-Best-Fit-Graph]].

## Converts To / From

- From: raw instrument readings
- To: [[Line-of-Best-Fit-Graph]]

## Related Quantities

- _Any measured quantity, e.g._ [[Acceleration]]

## Related Methods

- [[Significant-Figures-in-Measurements]]
- [[Calculating-Percentage-Uncertainty]]

## Common Mistakes

- Omitting the unit, or putting units next to every value instead of in the heading.
- Mixed decimal places within one column.
